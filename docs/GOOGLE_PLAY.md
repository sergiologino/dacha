# Публикация «Любимая Дача» в Google Play

Приложение — это PWA (веб-приложение). В Google Play его публикуют в виде обёртки **TWA (Trusted Web Activity)**: пользователь устанавливает небольшой Android-пакет, который открывает ваш сайт в полноэкранном режиме без панели браузера.

---

## 1. Что уже должно быть в приложении (чек-лист)

Перед сборкой под Play проверьте:

| Требование | Где проверить |
|------------|----------------|
| **Сайт по HTTPS** | Продакшен-домен (например `https://dacha-ai.ru`) открывается по HTTPS. |
| **Манифест PWA** | В проекте есть `public/manifest.json`: имя, `short_name`, `start_url`, `display`, `theme_color`, иконки. |
| **Иконки** | В `public/icons/` есть иконки 192×192 и 512×512 (в т.ч. `maskable` для Android). |
| **Service Worker** | Next.js при деплое отдаёт сервис-воркер для офлайн/пуша (если используете). |
| **Деплой** | Сайт уже развёрнут и доступен (см. [DEPLOY.md](./DEPLOY.md)). |

Если всё выше выполнено, приложение к публикации в Play с точки зрения кода готово. Дальше — только сборка обёртки TWA и настройка в Play Console.

---

## 2. Установка Bubblewrap

[Bubblewrap](https://github.com/GoogleChromeLabs/bubblewrap) — официальный инструмент для упаковки PWA в Android (TWA).

На компьютере с **Node.js 18+**:

```bash
npm install -g @bubblewrap/cli
```

При первом запуске Bubblewrap предложит скачать JDK и Android SDK — соглашайтесь, чтобы окружение настроилось само.

---

## 3. Создание Android-проекта (TWA)

Подставьте **реальный URL** вашего сайта и манифеста (должны быть доступны по HTTPS).

```bash
bubblewrap init --manifest=https://dacha-ai.ru/manifest.json
```

Интерактивно укажите:

1. **Package name** — уникальный ID приложения, например: `ru.dachaai.app` или `com.yourcompany.dacha`. Менять потом будет сложно, выберите сразу окончательный.
2. **Name** — название приложения в лаунчере (можно «Любимая Дача»).
3. **Launcher name** — короткое имя (например `Любимая Дача`).
4. **Theme color** — цвет интерфейса (в манифесте уже есть `#059669`, можно оставить или ввести в формате `#RRGGBB`).
5. **Background color** — фон загрузки (например `#ffffff`).
6. **Start URL** — с какой страницы открывать приложение. Должен совпадать с доменом сайта, например: `https://dacha-ai.ru/garden`.
7. **Signing key** — ключ для подписи:
   - Можно оставить значения по умолчанию (Bubblewrap создаст `android.keystore` в папке проекта).
   - Запомните пароли: они понадобятся при каждой сборке. **Ключ и пароли храните в безопасности и не выкладывайте в репозиторий.**

После завершения будет создана папка с Android-проектом (например `twa-manifest.json` и конфиг).

## 4. Первая сборка и загрузка в Play Console

### 4.1. Сборка AAB (рекомендуется для Play)

В папке, где выполняли `bubblewrap init`:

```bash
bubblewrap build
```

Введите пароль keystore при запросе. В результате получите **Android App Bundle** (`.aab`), например `app-release-bundle.aab` — его и нужно загружать в Google Play (для новых приложений с 2021 года обязателен именно AAB, не APK).

При необходимости можно собрать APK для ручной установки/тестов:

```bash
bubblewrap build --apk
```

### 4.2. Создание приложения в Play Console

1. Откройте [Google Play Console](https://play.google.com/console/).
2. **Создать приложение** → укажите название, язык по умолчанию, тип (приложение/игра), категорию и т.д.
3. Заполните обязательные разделы политики (Политика конфиденциальности, Правила и т.д. — консоль проведёт по чек-листу).
4. В разделе **Релизы** создайте релиз (например, внутреннее тестирование), загрузите собранный **AAB**.
5. После загрузки дождитесь обработки. Если включён **Play App Signing** (по умолчанию для новых приложений — да), Google подписывает приложение своим ключом. Этот ключ и нужен для следующего шага.

## 5. Digital Asset Links (обязательно для TWA)

Чтобы приложение открывалось как **полноэкранная** TWA (без панели браузера), сайт должен подтвердить связь с Android-приложением через файл **Digital Asset Links**.

### 5.1. Получить SHA-256 отпечаток

- Если используете **Play App Signing** (рекомендуется):
  - В Play Console: **Настройка приложения** → **Целостность приложения** (App integrity) → **Подписывание приложения**.
  - Скопируйте **SHA-256 отпечаток сертификата ключа подписывания приложения** (App signing key certificate). Именно он нужен для `assetlinks.json`.
- Для локальной установки (тест через `adb install`): используйте отпечаток **ключа загрузки** (Upload key) или получите его так:
  ```bash
  keytool -printcert -jarfile app-release-signed.apk | findstr SHA256
  ```
  (На macOS/Linux: `grep SHA256` вместо `findstr SHA256`.)

### 5.2. Файл assetlinks.json

Создайте на сайте файл по адресу:

**URL:** `https://ваш-домен/.well-known/assetlinks.json`

В этом проекте уже есть заготовка файла:

- **Файл:** `public/.well-known/assetlinks.json`

Замените в нём:

1. **package_name** — на тот, что указали в `bubblewrap init` (сейчас в шаблоне: `ru.dachaai.app`).
2. **sha256_cert_fingerprints** — на реальный SHA-256 отпечаток из Play Console (раздел «Целостность приложения» → ключ подписывания приложения). Формат: один или несколько отпечатков вида `AA:BB:CC:DD:...` (64 символа в hex, разделённые двоеточиями).

Пример итогового содержимого:

```json
[{
  "relation": ["delegate_permission/common.handle_all_urls"],
  "target": {
    "namespace": "android_app",
    "package_name": "ru.dachaai.app",
    "sha256_cert_fingerprints": [
      "AA:BB:CC:DD:EE:FF:00:11:22:33:44:55:66:77:88:99:AA:BB:CC:DD:EE:FF:00:11:22:33:44:55:66:77:88:99"
    ]
  }
}]
```

- **package_name** — тот же, что указывали в `bubblewrap init` (например `ru.dachaai.app`).
- **sha256_cert_fingerprints** — массив из одного или нескольких отпечатков. Для установки из Play Store подставьте отпечаток **ключа подписывания приложения** из Play Console. Для локальной установки — отпечаток вашего keystore (см. выше).

Файл должен отдаваться с заголовком `Content-Type: application/json`. Next.js отдаёт файлы из `public/` по пути `/.well-known/assetlinks.json`; для `.json` обычно тип выставляется автоматически. После деплоя откройте в браузере `https://dacha-ai.ru/.well-known/assetlinks.json` и убедитесь, что виден валидный JSON.

### 5.3. Проверка

- Установите приложение из внутреннего теста в Play Console (или собранный AAB/APK с тем же package name).
- Откройте приложение. Если Asset Links совпадают, интерфейс будет без адресной строки (полноэкранная TWA). Если нет — откроется как Custom Tab (с панелью браузера); тогда перепроверьте package name и SHA-256 в `assetlinks.json` и что файл доступен по HTTPS без редиректа.

## 6. Заполнение карточки в Play Console

Перед публикацией в продакшен нужно заполнить:

- **Описание приложения** (краткое и полное).
- **Скриншоты** — минимум по 2 для телефона (обязательно). При необходимости — для планшета.
- **Графическая обложка** (feature graphic) — 1024×500.
- **Иконка приложения** — 512×512 (можно взять из `public/icons/icon-512.png`).
- **Политика конфиденциальности** — URL страницы с текстом (обязательно, если собираете персональные данные или логин).
- **Рейтинг контента** — анкета и, при необходимости, вопрос о рекламе/платежах.
- **Целевая аудитория** — возрастная группа и т.д.

Консоль показывает чек-лист по каждому разделу; пройдите все пункты, чтобы разблокировать релиз.

---

## 7. Краткая последовательность

1. Убедиться, что сайт на HTTPS, манифест и иконки в порядке (раздел 1).
2. Установить Bubblewrap, выполнить `bubblewrap init` по URL манифеста, задать package name и ключ (разделы 2–3).
3. Собрать AAB: `bubblewrap build`, загрузить в Play Console во внутренний тест (раздел 4).
4. В Play Console взять SHA-256 ключа подписывания приложения, добавить `public/.well-known/assetlinks.json`, задеплоить сайт (раздел 5).
5. Проверить на устройстве: приложение открывается полноэкранно (TWA).
6. Заполнить описание, скриншоты, политику, рейтинг и выпустить релиз в продакшен (раздел 6).
