# План: Фича «Грядки — рассада, дата, фото растения»

## Цели
1. Тип грядки **«Рассада дома»**.
2. **Редактирование даты посадки** при добавлении и после.
3. **Фото растения на грядке**: клик по растению → иконка камеры → съёмка → фото привязывается к растению, грядке и дате.

---

## 1. Тип грядки «Рассада дома»

**Объём:** минимальный.

- **Данные:** тип в БД уже строка (`Bed.type`), новых миграций не нужно.
- **Фронт:** в `app/(app)/garden/page.tsx`:
  - В объекты `bedTypeLabels` и `bedTypeEmoji` добавить ключ `seedling_home`: «Рассада дома» и эмодзи (например 🏠 или 🌱).
  - В `<select>` при создании грядки добавить `<option value="seedling_home">… Рассада дома</option>`.

---

## 2. Редактирование даты посадки

**Объём:** бэкенд + фронт.

### 2.1 API
- **PATCH /api/plants** (новый метод в `app/api/plants/route.ts`):
  - Тело: `{ id: string, plantedDate?: string (ISO), name?: string, … }`.
  - Проверка: растение принадлежит текущему пользователю.
  - Обновить только переданные поля, вернуть обновлённый plant.

### 2.2 Хуки
- В `lib/hooks/use-plants.ts`:
  - Функция `updatePlant(data: { id: string; plantedDate?: string; name?: string })`.
  - Мутация `useUpdatePlant()` с инвалидацией `["plants"]` и `["beds"]`.

### 2.3 UI
- **При добавлении растения:** в форме «Добавить растение» в `BedCard` добавить поле даты (input type="date" или date picker), значение по умолчанию — сегодня. При вызове `createPlant.mutate` передавать `plantedDate` (ISO строка).
- **После добавления:** в строке растения показывать дату посадки; по клику на дату — редактирование (inline input type="date" или маленький popover с датой). По сохранению вызывать `updatePlant.mutate({ id, plantedDate })`.

---

## 3. Фото растения на грядке

**Объём:** API загрузки + хранение + UI (камера/выбор файла, привязка к растению/грядке/дате).

### 3.1 API
- **POST /api/photos** (новый файл `app/api/photos/route.ts`):
  - Принимать `FormData`: `file` (изображение), `plantId`, `bedId`, `takenAt` (опционально, ISO строка; по умолчанию — сейчас).
  - Проверка: пользователь авторизован, растение и грядка принадлежат ему.
  - Сохранение файла: в `public/uploads/` с уникальным именем (например `cuid().jpg`), при необходимости создать директорию.
  - Создать запись `Photo` с `url` (например `/uploads/xxx.jpg`), `plantId`, `bedId`, `takenAt`, `userId`.
  - Вернуть созданный объект Photo.
- **GET /api/beds:** в `include` для `plants` добавить `photos: { orderBy: { takenAt: 'desc' } }`, чтобы у каждого растения в ответе был массив `photos` (для превью и счётчика на карточке).

### 3.2 Хранилище
- Директория `public/uploads/` (создаётся при первой загрузке), в `.gitignore` добавить `public/uploads/`.
- В продакшене (Docker) запись на диск доступна; при переходе на serverless позже можно заменить на S3/Blob.

### 3.3 Хуки
- В `lib/hooks/use-beds.ts` (или новый `use-photos.ts`): функция `uploadPlantPhoto(formData)` и мутация `useUploadPlantPhoto()` с инвалидацией `["beds"]`.

### 3.4 UI
- В **BedCard**, в списке растений: у каждой строки растения добавить иконку камеры (например справа рядом с удалением). По клику:
  - Вариант A: открыть тот же паттерн, что на странице камеры — скрытый `<input type="file" accept="image/*" capture="environment">` + при необходимости превью и поле «Дата фото» (по умолчанию сегодня).
  - После выбора/съёмки: сформировать FormData (file, plantId, bedId, takenAt), вызвать `uploadPlantPhoto`, по успеху обновить список (React Query invalidation).
- Показ фото: под именем растения вывести миниатюры последних фото (например 1–2) или бейдж «N фото» со ссылкой/раскрытием. Данные уже придут в `plant.photos` после доработки GET /api/beds.

---

## Порядок реализации

1. **Тип грядки «Рассада дома»** — только UI, без API.
2. **Редактирование даты** — PATCH /api/plants → useUpdatePlant → форма добавления (дата по умолчанию) → редактирование даты в строке растения.
3. **Фото растения** — POST /api/photos + сохранение в public/uploads → расширение GET /api/beds (plant.photos) → useUploadPlantPhoto → в BedCard: иконка камеры у растения, выбор/съёмка, опционально дата фото, превью/счётчик фото.
